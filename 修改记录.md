# 修改记录

## 一、访客模式功能

### 1.1 功能概述

实现访客模式（游客模式），允许未登录用户浏览首页内容和观看视频，同时优化用户菜单的显示逻辑。

### 1.2 新增功能

- **访客模式开关** - 在后台管理的"站点配置"页面中添加"启用访客模式"开关
- **访客菜单** - 未登录用户显示简化的用户菜单，只显示"登录"按钮和版本信息
- **直播访客支持** - 访客可以访问直播页面和相关 API
- **登录状态切换** - 登录后自动清除访客状态，显示完整用户菜单

### 1.3 修改文件

| 文件路径 | 修改内容 |
|---------|---------|
| `src/proxy.ts` | 修改配置读取路径、添加直播路径、更新 middleware 规则 |
| `src/app/api/server-config/route.ts` | 添加 SiteConfig 内部请求支持 |
| `src/app/api/admin/site/route.ts` | 添加 AllowGuestBrowse 字段支持 |
| `src/app/api/login/route.ts` | 登录成功后清除访客 cookie |

## 二、自动填充 Key 功能

### 2.1 功能概述

在后台管理页面添加视频源和直播源时，输入名称后系统会自动根据名称生成 Key 并填充到 Key 输入框中。

**生成规则：**
- 中文字符：取拼音首字母（大写）
- 英文字符：取首字母（大写）
- 数字字符：保留原数字
- 示例："豆瓣" → "DB"，"爱奇艺" → "AQY"，"Netflix" → "N"

### 2.2 修改文件

| 文件路径 | 修改内容 |
|---------|---------|
| `src/app/admin/page.tsx` | 视频源和直播源名称输入框添加自动填充逻辑 |

### 2.3 详细修改

#### 2.3.1 视频源 Key 自动填充

原始代码：
```typescript
<input

  placeholder='名称'
  value={newSource.name}
```

修改后代码：
```typescript
<input
  type='text'
  placeholder='名称'
```

#### 2.3.3 核心函数：generateKeyFromName

```typescript
const generateKeyFromName = (name: string): string => {
  if (!name) return '';

  const initials = name
    .split('')
    .map((char) => {
      if (/[a-zA-Z]/.test(char)) return char.toUpperCase();
      if (/[0-9]/.test(char)) return char;
      const result = pinyin(char, { pattern: 'first' });
      return result || char;
    })
    .join('');

  return initials || name.substring(0, 4).toUpperCase();
};
```

说明：
- 使用 `pinyin-pro` 库将中文字符转换为拼音首字母
- 英文字符直接转大写
- 数字字符保持不变
- 如果转换失败，取前 4 个字符转大写作为备用方案

## 三、技术细节总结

### 3.1 Cookie 管理

| Cookie 名称 | 用途 | 设置时机 | 清除时机 |
|------------|------|---------|---------|
| `user_auth` | 用户认证信息 | 登录成功 | 登出 |
| `guest_mode` | 访客模式标识 | 登出（开启访客模式时） | 登录成功 |

### 3.2 配置存储

| 配置项 | 存储位置 | 字段名 | 默认值 |
|-------|---------|-------|-------|
| 访客模式开关 | `SiteConfig` | `AllowGuestBrowse` | `false` |

### 3.3 路径权限

**访客可访问路径：**
- `/` - 首页
- `/douban` - 豆瓣
- `/play` - 播放
- `/search` - 搜索
- `/shortdrama` - 短剧
- `/bilibili` - B站
- `/youtube` - YouTube
- `/live` - 直播

**访客可访问 API：**
- `/api/live/` - 直播 API

### 4.1 访客模式 Docker 环境协议错误

**问题描述：**
在 Docker 容器中，middleware 尝试通过 HTTPS 访问内部 API `https://0.0.0.0:3000/api/server-config`，但服务器内部是 HTTP 协议，导致 SSL 错误 `ERR_SSL_WRONG_VERSION_NUMBER`，访客模式无法正常工作。

**错误日志：**
```
[GuestMode] Error fetching config: TypeError: fetch failed
[cause]: [Error: ERR_SSL_WRONG_VERSION_NUMBER]
```

**修改文件：**
| 文件路径 | 修改内容 |
|---------|---------|
| `src/proxy.ts` | 添加 Docker 环境自动识别，使用 HTTP 协议进行内部请求 |

**原始代码：**
```typescript
async function getGuestBrowseConfig(request: NextRequest): Promise<boolean> {
  try {
    const url = new URL('/api/server-config', request.url);
    url.searchParams.set('key', 'SiteConfig');

    const response = await fetch(url.toString(), {
      headers: { 'x-internal-request': 'true' },
    });
    // ...
  }
}
```

**修改后代码：**
```typescript
async function getGuestBrowseConfig(request: NextRequest): Promise<boolean> {
  try {
    // 自动识别协议：Docker 容器内部使用 HTTP，外部使用原协议
    const url = new URL('/api/server-config', request.url);
    const isDocker = process.env.DOCKER_BUILD === 'true';
    
    if (isDocker) {
      // Docker 环境：使用 HTTP + localhost
      url.protocol = 'http:';
      url.hostname = 'localhost';
      url.port = '3000';
    }
    
    url.searchParams.set('key', 'SiteConfig');

    const response = await fetch(url.toString(), {
      headers: { 'x-internal-request': 'true' },
    });
    // ...
  }
}
```

**说明：**
- 通过 `process.env.DOCKER_BUILD` 环境变量判断是否在 Docker 环境中
- Docker 环境使用 `http://localhost:3000` 进行内部通信
- 非 Docker 环境保持原有协议和主机名

### 4.2 UserMenu 组件 Hydration 错误

**问题描述：**
React Hydration 错误，服务端和客户端渲染内容不一致。`isGuest` 初始值为 `false`，但客户端检测到 cookie 后变成 `true`，导致服务端渲染访客菜单和客户端渲染用户菜单不一致。

**修改文件：**
| 文件路径 | 修改内容 |
|---------|---------|
| `src/components/UserMenu.tsx` | 修改 `isGuest` 初始值，初始化时检查 cookie |

**原始代码：**
```typescript
const [isGuest, setIsGuest] = useState(false);
```

**修改后代码：**
```typescript
const [isGuest, setIsGuest] = useState(() => {
  // 初始化时检查 cookie，确保服务端和客户端一致
  if (typeof window !== 'undefined') {
    const guestCookie = document.cookie
      .split('; ')
      .find(row => row.startsWith('guest_mode='));
    return !!guestCookie && guestCookie.split('=')[1] === 'true';
  }
  return false;
});
```

**说明：**
- 使用函数式初始化，在组件挂载时立即检查 cookie
- 确保客户端首次渲染时 `isGuest` 状态与 cookie 一致
- 避免服务端和客户端渲染内容不一致导致的 Hydration 错误

### 4.3 HeroBanner 组件 React error #310

**问题描述：**
React error #310，表示 Hook 调用顺序或数量不一致。`useEffect` 在早期返回（`return null`）之前被调用，违反了 React Hooks 规则。

**错误日志：**
```
Uncaught Error: Minified React error #310
visit https://react.dev/errors/310
```

**修改文件：**
| 文件路径 | 修改内容 |
|---------|---------|
| `src/components/HeroBanner.tsx` | 将早期返回移到所有 Hook 之后 |

**原始代码：**
```typescript
// 使用滑动手势 Hook
const swipeHandlers = useSwipeGesture({
  onSwipeLeft: handleNext,
  onSwipeRight: handlePrev,
});

// 预加载背景图片
useEffect(() => {
  // ...
}, [items, currentIndex]);

if (!items || items.length === 0) {
  return null;
}

// 检查并刷新缺失的 trailer URL
useEffect(() => {
  // ...
}, [items, refreshedTrailerUrls, refreshTrailerUrl, enableVideo]);
```

**修改后代码：**
```typescript
// 使用滑动手势 Hook
const swipeHandlers = useSwipeGesture({
  onSwipeLeft: handleNext,
  onSwipeRight: handlePrev,
});

// 早期返回检查（必须在所有 Hook 之后）
if (!items || items.length === 0) {
  return null;
}

// 预加载背景图片
useEffect(() => {
  // ...
}, [items, currentIndex]);

// 检查并刷新缺失的 trailer URL
useEffect(() => {
  // ...
}, [items, refreshedTrailerUrls, refreshTrailerUrl, enableVideo]);
```

**说明：**
- React Hooks 规则：所有 Hook 必须在组件顶层无条件调用
- 早期返回（`return null`）必须在所有 Hook 调用之后
- 调整代码顺序，确保 Hook 调用顺序一致
